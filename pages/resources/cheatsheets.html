Web Security Cheatsheet

A comprehensive reference guide for web security best practices and common vulnerabilities.

OWASP Top 10

  • Injection: Use parameterized queries, input validation
  • Broken Authentication: Implement MFA, secure session management
  • Sensitive Data Exposure: Encrypt data in transit and at rest
  • XXE: Disable XML external entity processing
  • Broken Access Control: Implement proper authorization checks

Security Headers

Content-Security-Policy: default-src 'self'
Strict-Transport-Security: max-age=31536000
X-Frame-Options: DENY
X-Content-Type-Options: nosniff
X-XSS-Protection: 1; mode=block
Tip: Always implement security headers to protect against common web vulnerabilities.

Common Attacks Prevention

  • SQL Injection: Use prepared statements, input validation
  • XSS: Output encoding, CSP implementation
  • CSRF: Use CSRF tokens, SameSite cookies
  • Clickjacking: X-Frame-Options header
Warning: Regularly update your security knowledge and tools.

Network Security Cheatsheet

Essential network security commands and configuration guidelines.

Firewall Rules

# Allow SSH
iptables -A INPUT -p tcp --dport 22 -j ACCEPT

# Allow HTTP/HTTPS
iptables -A INPUT -p tcp --dport 80 -j ACCEPT
iptables -A INPUT -p tcp --dport 443 -j ACCEPT

# Default deny
iptables -P INPUT DROP
iptables -P FORWARD DROP
iptables -P OUTPUT ACCEPT

VPN Configuration

# OpenVPN server config
port 1194
proto udp
dev tun
ca ca.crt
cert server.crt
key server.key
dh dh2048.pem
server 10.8.0.0 255.255.255.0
push "redirect-gateway def1"
push "dhcp-option DNS 8.8.8.8"
keepalive 10 120
cipher AES-256-CBC
comp-lzo
user nobody
group nobody
persist-key
persist-tun
status openvpn-status.log
log-append openvpn.log
verb 3
Tip: Regularly audit your firewall rules and VPN configurations.

Network Commands

  • Port Scan: nmap -sV target
  • Network Monitor: tcpdump -i eth0
  • Route Check: traceroute target
  • DNS Lookup: dig domain
Note: Always use network monitoring tools to detect suspicious activity.

Linux Security Cheatsheet

Common Linux security commands and system hardening tips.

User Management

# Create user
useradd -m username
passwd username

# Add to sudo group
usermod -aG sudo username

# Check user permissions
id username
groups username

File Permissions

# Set permissions
chmod 755 directory
chmod 644 file

# Change ownership
chown user:group file

# Set ACL
setfacl -m u:user:rwx file
Tip: Follow the principle of least privilege when setting permissions.

System Hardening

  • Disable root login: Edit /etc/ssh/sshd_config
  • Enable firewall: ufw enable
  • Update system: apt update && apt upgrade
  • Check open ports: netstat -tulpn
Warning: Always backup your system before making security changes.
